# Handbook

> The single source of truth for the principles that guide my engineering work.

This handbook defines the principles that guide my engineering work.

It describes how I think, how I approach engineering, how I make decisions and how I communicate.

The purpose of this handbook is not to document technologies, products or current projects. It captures principles that should remain valid regardless of the tools, languages or platforms involved.

Every public artifact should be consistent with this handbook, including:

- Articles
- Talks
- Website
- Resume
- LinkedIn
- GitHub
- AI assisted content

This is a living document.

It is expected to evolve with experience, not with trends.

The goal is consistency, not perfection.

When in doubt, favor the handbook over optimization.

Identity comes first.

If something contradicts identity.md, don't fix the inconsistency.

Find out which one is wrong.
